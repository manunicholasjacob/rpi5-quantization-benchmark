SUMMARY OF CONTRIBUTION

This paper presents a statistically rigorous, fully reproducible benchmark of
INT8 post-training quantization (PTQ) on Raspberry Pi 5 (ARM Cortex-A76) using
ONNX Runtime 1.18.0. The work directly addresses a practical gap: while INT8
quantization is widely recommended for edge CPU inference, practitioners have no
statistically validated, hardware-specific characterization of when it helps and
when it introduces deployment instability.

The three principal findings are:

1. LATENCY AND COMPRESSION: INT8 delivers 3.80–11.73× P50 latency reduction
   (p < 0.001, Cohen's d > 100 in all four model×dataset configurations) with
   3.3–4.0× model size compression, validated with 300-iteration trials and
   95% confidence intervals.

2. TAIL-LATENCY FAILURE MODE (key novelty): ResNet-18 INT8 on CIFAR-100
   exhibits a 1.94× P90/P50 tail-latency ratio—a bimodal distribution driven
   by stochastic LLC cache pressure from 108 Q/DQ cast operations wrapping
   large (2.3M-element) weight tensors. Critically, the same model on SVHN
   shows no tail inflation (P90/P50 = 1.03), demonstrating that the hazard is
   dataset-conditioned. This failure persists across batch sizes 1–8 and is
   completely invisible in median-only benchmarks—representing a concrete
   deployment hazard for real-time SLO-constrained systems that has not been
   previously documented.

3. ENERGY-DELAY PRODUCT: Surrogate-modeled EDP improvements of 5.2× (MV3-S)
   and 43.3× (R18), with explicit uncertainty quantification (±20–60%).
